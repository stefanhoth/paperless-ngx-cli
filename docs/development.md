# Development

## Requirements

- Go 1.21+
- Docker (for schema generation without a running Paperless instance)
- A running Paperless-NGX 2.x instance (optional — only needed for `make generate`)

## Building

```bash
make build          # compiles ./paperless
make install        # installs to /usr/local/bin/paperless
go test ./...       # run unit tests
```

## Regenerating the API Client

The generated client lives in `api/paperless.gen.go` and is built with
[oapi-codegen](https://github.com/oapi-codegen/oapi-codegen) from the
Paperless-NGX OpenAPI schema. Regenerate whenever Paperless ships a new version:

```bash
# No running instance needed — pulls the official Docker image and exports the schema:
make generate-docker VERSION=v2.20.15
make build

# Or, if you have a running Paperless instance:
make generate   # uses PAPERLESS_URL + PAPERLESS_API_TOKEN
make build
```

`scripts/fix-schema.py` patches two known issues in the upstream schema before generation:
- Renames response types that conflict with oapi-codegen's generated wrapper names
- Fixes `Tag.children` — the API returns objects, but the schema declares integer IDs

A daily CI workflow (`schema-check.yml`) detects new Paperless releases and opens a GitHub
issue when the schema needs updating.

## Releasing

Releases are fully automated via GoReleaser and GitHub Actions (`release.yml`).

```bash
git tag v1.2.3
git push origin v1.2.3
```

This triggers:
1. GoReleaser builds binaries for linux/darwin × amd64/arm64 and creates a GitHub Release
2. `orhun/git-cliff-action` regenerates `CHANGELOG.md` and commits it back to `main`
3. GoReleaser pushes an updated Homebrew formula to [stefanhoth/homebrew-tap](https://github.com/stefanhoth/homebrew-tap)

### Homebrew Tap Setup

The tap lives at [github.com/stefanhoth/homebrew-tap](https://github.com/stefanhoth/homebrew-tap).
GoReleaser manages the formula — you never need to edit it manually.

**Required secret** in this repo (`Settings → Secrets → Actions`):

| Secret | Value |
|---|---|
| `HOMEBREW_TAP_GITHUB_TOKEN` | A GitHub PAT with `Contents: Read & Write` permission scoped to `stefanhoth/homebrew-tap` |

**Creating the PAT:**
1. GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens
2. Resource owner: `stefanhoth`, Repository access: `Only select repositories` → `homebrew-tap`
3. Permissions: `Contents` → `Read and write`
4. Copy the token and add it as `HOMEBREW_TAP_GITHUB_TOKEN` in `stefanhoth/paperless-ngx-cli` secrets

**Tap repo layout** (GoReleaser creates/updates this automatically):

```
homebrew-tap/
└── Formula/
    └── paperless-ngx-cli.rb   # auto-generated by GoReleaser on each release
```

**Installing via Homebrew** (after first release with tap configured):

```bash
brew tap stefanhoth/tap
brew install paperless-ngx-cli
```

## API Versioning

The CLI sends `Accept: application/json; version=N` with every request. `APIVersion` in
`cmd/root.go` is the single source of truth. When Paperless ships a new stable API version,
bump `APIVersion`, update the Version Support table in the README, and release a new major
CLI version.

See the [Version Support](../README.md#version-support) section in the README for the
version compatibility matrix.
